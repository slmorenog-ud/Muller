\begin{abstract}
    El presente artículo expone el análisis teórico, la implementación computacional y la evaluación experimental del Método de Müller para la búsqueda de raíces de ecuaciones no lineales. Este método, propuesto por David Müller en 1956, extiende la idea del método de la secante al ajustar una parábola a través de tres puntos, obteniendo una tasa de convergencia superlineal de aproximadamente 1.84. A diferencia de métodos clásicos, es capaz de hallar raíces complejas partiendo de aproximaciones iniciales reales. Se presentan la derivación completa de las fórmulas, el análisis de estabilidad numérica, cinco casos de estudio representativos y una implementación robusta en Python.
\end{abstract}

\section{Introducción}
La resolución de ecuaciones no lineales de la forma $f(x) = 0$ es uno de los problemas más recurrentes en las ciencias de la ingeniería y la física matemática. Desde el cálculo de corrientes en circuitos no lineales hasta la determinación de frecuencias naturales en estructuras, encontrar las raíces de una función es una tarea que rara vez admite soluciones analíticas cerradas.

Los métodos numéricos constituyen la herramienta esencial para abordar este problema. Mientras que el método de Newton-Raphson ofrece una convergencia cuadrática, exige el conocimiento de la derivada, lo cual no siempre es posible. El método de la secante elimina esta restricción pero se limita a raíces reales. En este contexto, el método de Müller surge como una generalización potente: utiliza interpolación parabólica para capturar la curvatura local, logrando una convergencia más rápida que la secante y la capacidad única de navegar hacia el plano complejo.

El propósito de este proyecto es delimitar los fundamentos matemáticos de Müller, implementar un algoritmo eficiente y demostrar su versatilidad a través de la resolución detallada de problemas de diversa naturaleza.

\section{Desarrollo Teórico}

\subsection{Intuición Geométrica}
La idea central es construir una parábola $y = a(x-p_2)^2 + b(x-p_2) + c$ que pase por tres puntos iniciales $(p_0, f(p_0))$, $(p_1, f(p_1))$ y $(p_2, f(p_2))$. La raíz de esta parábola que se encuentre más próxima al punto $p_2$ se define como la siguiente aproximación $p_3$. Esta curvatura permite al algoritmo "doblarse" siguiendo la forma de la función, facilitando el hallazgo de raíces complejas incluso partiendo de valores reales (cuando el discriminante de la parábola es negativo).

\subsection{Derivación de las Fórmulas}
Consideramos el cambio de variable $t = x - p_2$. El polinomio cuadrático es $P(t) = at^2 + bt + c$. Evaluando en $t=0$ (donde $x=p_2$), obtenemos $c = f(p_2)$. Para hallar $a$ y $b$, definimos $h_0 = p_0 - p_2$, $h_1 = p_1 - p_2$, $e_0 = f(p_0) - f(p_2)$ y $e_1 = f(p_1) - f(p_2)$. Resolviendo el sistema por eliminación o regla de Cramer:
\begin{equation}
    a = \frac{e_0 h_1 - e_1 h_0}{h_1 h_0^2 - h_0 h_1^2}, \quad b = \frac{e_1 h_0^2 - e_0 h_1^2}{h_1 h_0^2 - h_0 h_1^2}
\end{equation}

\subsection{Estabilidad Numérica}
Para evitar errores de redondeo por cancelación sustractiva cuando $b \approx \sqrt{b^2 - 4ac}$, se utiliza la fórmula cuadrática alternativa:
\begin{equation}
    z = \frac{-2c}{b \pm \sqrt{b^2 - 4ac}}
\end{equation}
Se elige el signo que maximice la magnitud del denominador. La nueva aproximación será $p_3 = p_2 + z$.

\section{Ejemplos y Problemas Resueltos}

\subsection{Problema 1: Función Polinomial}
$f(x) = x^3 - x - 1$. Con $p_0=0, p_1=1, p_2=2$.
Tras calcular $a=3, b=9, c=5$, obtenemos $z \approx -0.7362$, resultando en $p_3 \approx 1.2638$. El método converge a $x \approx 1.3247$ en solo 4 iteraciones.

\subsection{Problema 2: Función Trascendente (Constante Omega)}
$f(x) = e^{-x} - x$. Con $p_0=0, p_1=0.5, p_2=1$.
El método captura la caída exponencial y converge rápidamente a $x \approx 0.5671$.

\subsection{Problema 3: Función Trigonométrica}
$f(x) = \sen(x) - x/2$. Con $p_0=1, p_1=2, p_2=3$.
Esta función, relevante en oscilaciones, converge a $x \approx 1.8954$ en 4 iteraciones.

\subsection{Problema 4: El Poder de lo Invisible (Raíces Complejas)}
$f(x) = x^2 + 1$. Con $p_0=0, p_1=1, p_2=2$.
Aunque los puntos iniciales son reales, el discriminante $\sqrt{4-20} = 4i$ introduce la componente imaginaria. El método converge a $x = i$, demostrando su capacidad única.

\subsection{Problema 5: Función Mixta Logarítmica}
$f(x) = \ln(x) + x^2 - 5$. Con $p_0=1, p_1=2, p_2=3$.
El método converge en 3 iteraciones a $x \approx 2.0931$.

\section{Algoritmo e Implementación}
Se implementó en Python utilizando el módulo \texttt{cmath} para soporte nativo de aritmética compleja.

\begin{lstlisting}[language=Python, caption=Núcleo del Algoritmo de Müller]
import cmath

def muller(f, p0, p1, p2, tol=1e-10, max_iter=100):
    for _ in range(max_iter):
        h0, h1 = p0 - p2, p1 - p2
        f0, f1, f2 = f(p0), f(p1), f(p2)
        e0, e1 = f0 - f2, f1 - f2
        
        det = h0 * h1 * (h0 - h1)
        if det == 0: return p2
        
        a = (e0 * h1 - e1 * h0) / det
        b = (e1 * h0**2 - e0 * h1**2) / det
        c = f2
        
        disc = cmath.sqrt(b**2 - 4 * a * c)
        den = b + disc if abs(b + disc) >= abs(b - disc) else b - disc
        z = -2 * c / den if den != 0 else 0
        p3 = p2 + z
        
        if abs(z) < tol: return p3
        
        # Estrategia de actualización por cercanía
        points = sorted([p0, p1, p2], key=lambda p: abs(p - p3))
        p0, p1, p2 = points[0], points[1], p3
    return p2
\end{lstlisting}

\section{Conclusión}
El Método de Müller constituye una herramienta poderosa y flexible. A través de este proyecto, hemos demostrado que supera a la secante en velocidad de convergencia (1.84 vs 1.61) y posee la capacidad única de hallar raíces complejas sin requerir derivadas. La estabilidad numérica proporcionada por la fórmula cuadrática alternativa es fundamental para su éxito. En conclusión, Müller es una pieza esencial en la ingeniería numérica, ideal para sistemas donde la curvatura y las raíces imaginarias tienen un significado físico profundo.
